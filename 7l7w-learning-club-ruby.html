7L7W Learning Club - Ruby

By Lorena Mesa
on 11.03.2016

Some of my work mates suggested we have a learning club to pursue etc interests, a space for cross pollination of knowledge. First up for the learning club we’re trying our hand at Bruce Tate’s Seven Languages in Seven Weeks (or 7L7W for short).

The first language in 7L7W is Ruby.

Tate compares Ruby to Mary Poppins because like “Mary Poppins [who] made the household more efficient by making it fun ... Ruby does the same ... with more syntactic sugar than a spoonful”.

There are many ways to do one thing in Ruby

A few years ago when I first learned Ruby I stumbled on a great blog post comparing and contrasting the enumerable methods select, reject, collect, inject, and detect. Coming from R and Python I felt a bit overwhelmed by all the options for what seemed to be fairly comparable behaviors. According to Tate -- that’s the point; in Ruby there are many ways to do one thing.

One thing Tate emphasizes is the love that Matz, the creator of Ruby, has for languages like Perl and Lisp. Like Perl’s TMTOWTDI Ruby channels this idea with it’s syntactic sugar.

Let’s consider an example from 7L7W, the way you can define instance variables within a class. 

class Cookie
    attr_accessor :type, :ingredients
    def initialize(type, ingredients=[])
        @type = type
        @ingredients = ingredients
    end
end

An attr_accessor will create a setter and a getter method for each of the specified instance variables. Of course you can also do this:

class Cookie

    def initialize(type, ingredients=[])
        @type = type
        @ingredients = ingredients
    end

    def type=(type)  # Use an = in the method name
        @type = type
    end

    def type
        type
    end

    def set_ingredients(ingredients=[])   # Use own name!
        @ingredients = ingredients
    end

    def get_ingredients
       @ingredients
    end

end

Notice that there isn’t a mandated singular way of how to create the getter and setter methods nor is the naming of those methods.

DSLs and Metaprogramming

Writing code that is easy to read, that’s one of biggest wins with Ruby. You can also use this to allow to build DSLs - domain specific languages - like Rake. One way to think of a DSL is that it is a tool that “reads just like English”. Again we come back to the idea that Ruby aims to be human readable. The tradeoff here is while Ruby may not be as performant as a static language, it compensates by allowing a coder write code that is logical for humans to follow. It’s efficiency is its trust in the Ruby coder!

We can go farther down the path of what domain specific language lets us do by looking at how classes work in Ruby. It is legal in Ruby to open a build in Ruby class and modify or extend its source code. Ruby classes are open!

How can I see this? Well let’s say I really want each word in my strings to start with a capital letter. I could do this:

class String
    def to_s
        self.split.map(&:capitalize).join(' ')
    end
end

> “HELLO wORLD”.to_s
  “Hello World”

Metaprogramming in Ruby allows me to customize Ruby to my particular needs. While this can be dangerous if taken too far and cannot be generalizable for those with other needs (AKA weird bugs you don’t understand), it goes to show just how much Ruby aims to unleash the creative power of the coder.

Ruby - a perfect first language to learn?

Our 7L7W discussion came full circle to how my appreciation for Ruby have grown over time. Two of the learning club participants, myself included, took less traditional paths to make our way into web development. Ruby was a language we learned early on to jump start that path.

My original understanding of Ruby was so tied into Rails that I didn’t dig into Ruby and how it worked during that time. Not that a beginning coder would (arguably) have success at that. Now as a coder who doesn’t use Ruby except to mentor beginners, I can have come to see how Ruby as a language is like a onion. As you learn you can roll back each layer to learn more and more. It is a language the invites curiosity. You can tinker, compare and contrast implementations, or make your own!

Other scripting, dynamic languages of course have made learning to code more efficient (if a significant way you measure efficiency is in time). The Zen of Python after all says “readability counts” and goes so far as to enforce whitespace as a part of its syntax.

When all's said and done, Ruby is simply put a fun language. What metaprogramming hacks have you used, be it rewarding or not, in Ruby?
